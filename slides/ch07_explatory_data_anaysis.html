探索的データ分析(EDA)

  1. データに対する仮説を作る
  2. 可視化したり集計したりして仮説を確かめる
  3. 仮説を改善したり別の仮説を検討したりする

というサイクルを回す分析

準備

library(tidyverse)

実際に使うのはggplot2とdplyr

データへの質問

『緻密だが間違った質問に対する厳密な答えよりも、曖昧な正しい質問に対するそれなりな答えの方が遥かに優れている。』 John Tukey

原則

  • 仮説の質を上げるために仮説の量を増やす

フレームワーク

  • 変数の変動(variation)はどうなっているか？
  • 変数の間の共変動(covariation)はどうなっているか？

用語の整理

  • 変数 あなたが測定できる量、質、性質など(列)
  • 値 測定された変数の状態 (表に入っている値)
  • 観測 同一の条件下での測定値のあつまり（行）
  • 表データ 観測、変数で整理された値の集まり

詳しいことはchapter 12 Tidy dataでやります

パターン

データの中にパターンを見つけることができたとき次のことを考える。

  • このパターンは偶然ではないか？
  • パターンの意味を説明できるか？
  • パターンの強さはどのくらいか？
  • その関係には他の変数が影響していないか？
  • データを部分集合にしたときにもそのパターンは残るか？

パターンとモデル

なぜパターンはありがたいのか

パターンは共変動をあらわにするから

ワークフロー

  1. EDAでパターンを見つける
  2. パターンをモデル化する
  3. モデルを使って変動を減らす

例

疑問

diamondsでcutが良いほうがpriceが高い傾向が観察された。なぜ？

パターン

caratという中間変数が作用している。 すなわちcaratが大きい方がpriceが高くなる(パターン)、かつcutが悪い方がcaratが大きいためでは？

方針

このパターンを使ってcutとpriceの間の共変動を減らす。 これによりcutとpriceとの真の関係をあらわにする。

モデル化

ggplot(data = diamonds2, mapping = aes(x = log2(carat), y = log2(price))) +
  geom_point(alpha = 1/2) +
  geom_smooth(method = "lm")
## `geom_smooth()` using formula 'y ~ x'

cutとモデル残差

library(modelr) 
mod <- lm(log(price) ~ log(carat), data = diamonds) 
diamonds %>% 
  add_residuals(mod) %>% 
  mutate(resid = exp(resid)) %>%
  ggplot() +
    geom_boxplot(mapping = aes(x = cut, y = resid))

ggplot2 call

ggplot(data = faithful, mapping = aes(x = eruptions)) + 
  geom_freqpoly(binwidth = 0.25)

実は

ggplot(faithful, aes(eruptions)) + 
  geom_freqpoly(binwidth = 0.25)
